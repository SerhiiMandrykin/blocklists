## ğŸ“‚ Project Structure

This project uses "Infrastructure as Code" to manage blocklists.
* **`lists/`**: Source files. This is where I manually add domains (e.g., `adware.txt`, `garbage.txt`).
* **`configs/`**: JSON configuration files that tell the compiler how to merge the lists.
* **`combined/`**: **Do not edit manually.** These files are auto-generated by GitHub Actions.
